# BTS7960 Support Design

**Date:** 2026-03-25

**Goal:** Add first-class support for dual BTS7960 motor drivers while preserving the existing PWM+DIR driver path.

## Context

The current controller assumes one PWM pin and one direction pin per motor. BTS7960 uses two PWM inputs per motor (`RPWM`, `LPWM`) plus enable lines (`REN`, `LEN`). The repository already has drive mixing, ramping, and tests around the `set_speed(-1.0..1.0)` interface, so the new driver should fit behind the same interface rather than changing higher-level control flow.

## Decision

Use a dedicated `BTS7960MotorDriver` class selected through configuration.

## Why This Approach

1. Keeps the existing `DifferentialDrive` API unchanged.
2. Preserves backward compatibility for the current PWM+DIR driver.
3. Makes BTS7960 behavior testable in isolation.
4. Allows separate or shared enable pins through configuration without special-case code in control flow.

## Control Model

For each BTS7960 motor:

1. `REN` and `LEN` are configured as outputs and enabled during initialization.
2. Positive speed drives `RPWM` with duty cycle and forces `LPWM` to `0`.
3. Negative speed drives `LPWM` with duty cycle and forces `RPWM` to `0`.
4. Zero speed forces both PWM outputs to `0`.
5. Inversion flips the sign before mapping to `RPWM`/`LPWM`.

This matches the common BTS7960 usage pattern and supports the current bench wiring where both enable pins can be tied together externally or configured to the same GPIO.

## Configuration

Add a driver-type switch:

- `MOTOR_DRIVER_TYPE=PWM_DIR|BTS7960`

Keep existing `MOTOR1_*`/`MOTOR2_*` variables for the PWM+DIR path.

Add BTS7960-specific pins:

- `LEFT_MOTOR_RPWM`
- `LEFT_MOTOR_LPWM`
- `LEFT_MOTOR_REN`
- `LEFT_MOTOR_LEN`
- `RIGHT_MOTOR_RPWM`
- `RIGHT_MOTOR_LPWM`
- `RIGHT_MOTOR_REN`
- `RIGHT_MOTOR_LEN`

Default dual-BTS7960 mapping:

- Left: `18`, `13`, `23`, `24`
- Right: `12`, `16`, `20`, `21`

## Risks

1. Existing tests may assume direct construction of `MotorDriver` in `main.py`.
2. Shared enable pins must still work when `REN` and `LEN` are configured to the same GPIO.
3. Documentation currently describes the old PWM+DIR topology and must be updated consistently.

## Validation

1. Unit tests for `BTS7960MotorDriver` init, forward, reverse, stop, inversion.
2. Config tests for the new driver type and env overrides.
3. Compose and env docs include new variables.
4. Existing drive-mixing tests continue to pass with the common `set_speed` interface.
