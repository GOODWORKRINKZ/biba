# Target: BIBA_F103_REV_A

Custom BiBa driver PCB (prototype revision A) built around an
STM32F103C8T6. **Nothing has been soldered yet** — this target exists so
the PCB revision can be routed against a locked pinout.

## Why this pinout looks different from BLUEPILL_F103C8

BiBa has no buzzer. Audible feedback (voice / melodies / BLHeli-style
tones) is generated by modulating the BTS7960 wheel PWMs, and that needs
**four independent carrier frequencies at the same time**. On an
STM32F103C8 the only way to get four independent PWM carriers is one
hardware timer per line, so each of the four motor PWM lines is wired
to a different timer:

| Line      | Pin  | Timer / Channel | AF                       |
| --------- | ---- | --------------- | ------------------------ |
| `L_RPWM`  | PA8  | `TIM1_CH1`      | default AF               |
| `L_LPWM`  | PA15 | `TIM2_CH1`      | TIM2 partial remap 1     |
| `R_RPWM`  | PB1  | `TIM3_CH4`      | default AF               |
| `R_LPWM`  | PB9  | `TIM4_CH4`      | default AF               |

Partial-remap-1 for TIM2 routes `CH1 → PA15` and `CH2 → PB3`, but only
`CH1` is configured for AF (the PB3 pin is kept as GPIO for `L_REN`).
Full TIM2 remap is not used because that would grab PB10/PB11 which are
already owned by USART3/CRSF.

In traction mode the HAL points all four timers at the same
`BIBA_PWM_FREQUENCY_HZ` carrier and only writes the compare registers,
so the existing BTS7960 drive path is byte-compatible with a
one-timer board. `BIBA_TARGET_HAS_PER_CHANNEL_TIMER_PWM` is `1`.

## Other deltas from BLUEPILL_F103C8

| Area             | BLUEPILL_F103C8                 | BIBA_F103_REV_A                        |
| ---------------- | ------------------------------- | -------------------------------------- |
| Enable pins      | PB3 / PB4 / PB5 / PB8           | same (JTAG released)                   |
| MODE_SEL         | PB9                             | PA7 (PB9 is now TIM4_CH4)              |
| IMU INT1         | PB2                             | PB2 (unchanged)                        |
| Status LED       | PC13 (active low)               | PC13 (active low)                      |
| Current sense    | pull-down, 8.5 A/V, 0.1 V bias  | op-amp, 10 A/V, 0 V bias               |
| Battery divider  | 1 : 11 (~33 V max)              | 1 : 20 (~66 V max)                     |
| Default I-limit  | 18 A per side                   | 25 A per side                          |
| PA5 on ADC       | 12 V rail tap                   | chassis NTC                            |
| PA9..PA11        | TIM1_CH2..CH4 (PWM)             | no-connect (PWM moved to TIM2/3/4)     |
| Buzzer / tone    | "aux tone" on PA15 (TIM2_CH1)   | **removed — audio rides on the wheel PWM** |

## Caveats

- JTAG is released at boot so PB3/PB4/PA15 are usable as GPIO/AF — SWD
  stays available on PA13/PA14 for programming and debug.
- TIM1 is the only advanced-control timer: only it has hardware
  dead-time generation (BDTR). We do not rely on that — in traction
  mode the HAL's `set_channel_pair` helper forces the non-selected side
  to `CCR=0` so shoot-through is impossible by construction, and in
  audio mode the L and R PWM lines of the same side run at *different*
  frequencies, not as a complementary pair.
- Inter-motor edge alignment is lost in the new layout (four
  independent counters). BTS7960 does not care.
- USART1 on PA9/PA10 is still unusable for anything else — those pins
  are now plain GPIO no-connects.

## Adding more targets

Copy this directory, rename it to `<YOUR_TARGET>` (`SCREAMING_SNAKE`),
edit `target.h` and `target_config.h`, then register the target with a
single stanza in `firmware/platformio.ini` — see
`targets/README.md` for the template.
