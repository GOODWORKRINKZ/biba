# Таргет: BIBA_F103_REV_A

Кастомная плата управления BiBa (прототип ревизии A) на базе
STM32F103C8T6. **Пока ничего не распаяно** — таргет существует, чтобы
ревизию платы можно было разводить против фиксированной распиновки.

## Почему распиновка отличается от BLUEPILL_F103C8

В BiBa нет пищалки. Звуковая обратная связь (голос, мелодии,
BLHeli-style тоны) генерируется модуляцией PWM-сигналов на BTS7960, а
для этого нужно **четыре независимых несущих частоты одновременно**.
На STM32F103C8 единственный способ получить четыре независимых PWM —
это один аппаратный таймер на линию, поэтому каждая из четырёх
PWM-линий мотора сидит на своём таймере:

| Линия     | Пин  | Таймер / Канал  | AF                       |
| --------- | ---- | --------------- | ------------------------ |
| `L_RPWM`  | PA8  | `TIM1_CH1`      | AF по умолчанию          |
| `L_LPWM`  | PA15 | `TIM2_CH1`      | partial remap 1 для TIM2 |
| `R_RPWM`  | PB1  | `TIM3_CH4`      | AF по умолчанию          |
| `R_LPWM`  | PB9  | `TIM4_CH4`      | AF по умолчанию          |

Partial remap 1 для TIM2 раскидывает `CH1 → PA15` и `CH2 → PB3`, но в
AF настроен только `CH1` (PB3 остаётся обычным GPIO под `L_REN`).
Полный remap TIM2 не используется — он бы захватил PB10/PB11, которые
уже заняты USART3/CRSF.

В тяговом режиме HAL ставит на все четыре таймера одну и ту же
несущую `BIBA_PWM_FREQUENCY_HZ` и пишет только compare-регистры,
поэтому существующий код управления BTS7960 побайтово совместим с
платой на одном таймере. Флаг
`BIBA_TARGET_HAS_PER_CHANNEL_TIMER_PWM` равен `1`.

## Прочие отличия от BLUEPILL_F103C8

| Область           | BLUEPILL_F103C8                  | BIBA_F103_REV_A                          |
| ----------------- | -------------------------------- | ---------------------------------------- |
| Enable-пины       | PB3 / PB4 / PB5 / PB8            | то же (JTAG отключён)                    |
| MODE_SEL          | PB9                              | PA7 (PB9 теперь TIM4_CH4)                |
| IMU INT1          | PB2                              | PB2 (без изменений)                      |
| Status LED        | PC13 (active-low)                | PC13 (active-low)                        |
| Токовый сенсор    | pull-down, 8.5 А/В, смещение 0.1 В | оп-усил, 10 А/В, смещение 0 В          |
| Делитель батареи  | 1 : 11 (~33 В макс.)             | 1 : 20 (~66 В макс.)                     |
| Лимит тока (def.) | 18 А на сторону                  | 25 А на сторону                          |
| PA5 на ADC        | отвод 12 В                       | NTC шасси                                |
| PA9..PA11         | TIM1_CH2..CH4 (PWM)              | no-connect (PWM ушёл на TIM2/3/4)        |
| Бузер / тон       | «aux tone» на PA15 (TIM2_CH1)    | **убран — звук идёт через PWM колёс**    |

## Полная распиновка

| Назначение                                 | Пин         |
| ------------------------------------------ | ----------- |
| TIM1_CH1 — Left  RPWM                      | PA8         |
| TIM2_CH1 — Left  LPWM (partial remap 1)    | PA15        |
| TIM3_CH4 — Right RPWM                      | PB1         |
| TIM4_CH4 — Right LPWM                      | PB9         |
| Левый  BTS7960 R_EN / L_EN                 | PB3 / PB4   |
| Правый BTS7960 R_EN / L_EN                 | PB5 / PB8   |
| ADC1 IN0..IN3 — 4× `IS` BTS7960            | PA0..PA3    |
| ADC1 IN4 — VBAT (делитель 1 : 20)          | PA4         |
| ADC1 IN5 — NTC шасси                       | PA5         |
| ADC1 IN6 — ток шины                        | PA6         |
| USART3 TX / RX — CRSF                      | PB10 / PB11 |
| SPI2 NSS / SCK / MISO / MOSI               | PB12..PB15  |
| DATA_READY → SBC                           | PA12        |
| MODE_SEL (pull-up; GND = companion)        | PA7         |
| I2C1 SCL / SDA — IMU                       | PB6 / PB7   |
| IMU INT1                                   | PB2         |
| Status LED (active-low)                    | PC13        |
| SWD                                        | PA13 / PA14 |
| **Резерв / no-connect**                    | PA9, PA10, PA11, PB0 |

## Особенности

- JTAG отключается на старте, поэтому PB3/PB4/PA15 доступны как
  GPIO/AF — SWD остаётся на PA13/PA14 для программирования и отладки.
- TIM1 — единственный advanced-control-таймер из набора, и только у
  него есть аппаратная генерация dead-time (BDTR). Мы на это не
  полагаемся: в тяговом режиме хелпер `set_channel_pair` форсирует
  неактивную сторону в `CCR=0` — shoot-through по построению
  невозможен; а в audio-режиме L и R PWM одной стороны идут на
  *разных* частотах и не образуют комплементарной пары.
- Кросс-моторное выравнивание фронтов в новой раскладке потеряно
  (четыре независимых счётчика). BTS7960 это не волнует.
- USART1 на PA9/PA10 по-прежнему ни для чего не пригоден — теперь это
  просто GPIO no-connect.

## Как добавить ещё таргеты

Скопируй эту директорию, переименуй в `<YOUR_TARGET>`
(`SCREAMING_SNAKE`), отредактируй `target.h` и `target_config.h`,
затем зарегистрируй таргет одной секцией в
`firmware/platformio.ini` — шаблон смотри в `targets/README.md`.
