**Smoke test 1 — Default config (все тумблеры=1) — должен вести себя как до Phase 12:**

1. Прошей RP2040 firmware собранной с default config:
   ```bash
   cd firmware && pio run -e rpico_rp2040_standalone -t upload 2>&1
   ```
   (или перетащи firmware.uf2 на RP2040 в режиме USB MSC)

2. Подключи пульт (ELRS TX должен быть включен и bound к приёмнику)
3. Включи питание робота (аккумулятор)
4. Проверь:
   - При старте играет startup melody (если MELODY=1)
   - CH5 (arm) в положении DISARMED — моторы не крутятся
   - Переведи CH5 в ARMED:
     - Дай газ — робот едет вперёд с PI-регуляцией оборотов (RPM closed-loop)
     - Руль — робот поворачивает с дифференциалом
     - При neutral steering и throttle — моторы останавливаются
     - Heading hold CH10 в STABILIZED — робот пытается держать курс (коррекция заметна при боковом толчке)
   - Выключи пульт — failsafe: моторы останавливаются в течение ≤500 мс
   - При разарме (CH5 LOW) — моторы гарантированно остановлены (PWM zero)

5. Подключи USB-кабель, открой serial monitor (115200 baud):
   ```bash
   python3 -c "import serial; s=serial.Serial('/dev/ttyACM0',115200,timeout=0.5); [print(s.readline().decode(errors='replace').strip()) for _ in range(20)]"
   ```
   Проверь: нет аварийных сообщений (ERROR, HARD FAULT, stack overflow), видна нормальная телеметрия.

**Smoke test 2 — Open-loop config (BIBA_FEATURE_RPM_CLOSED_LOOP=0) — регрессионный дым-тест:**

1. Собери open-loop прошивку:
   ```bash
   cd firmware && pio run -e rpico_rp2040_standalone \
     --build-flags="-DBIBA_FEATURE_RPM_CLOSED_LOOP=0" 2>&1 | tail -5
   ```
2. Прошей её на RP2040 (drag-and-drop firmware.uf2)
3. Проверь:
   - При ARMED + газ: робот ЕДЕТ (duty идёт напрямую с mixer, без PI-регуляции)
   - Поведение: моторы реагируют на газ мгновенно, без ramp (ramp тоже отключён при RPM_CLOSED_LOOP=0)
   - Руль работает (дифференциальный микс)
   - Failsafe: моторы останавливаются при выключении пульта
   - **Критично:** робот должен быть УПРАВЛЯЕМ — не должно быть залипания моторов, неконтролируемого разгона, или полного отсутствия реакции на пульт
4. Верни default прошивку после теста:
   ```bash
   cd firmware && pio run -e rpico_rp2040_standalone -t upload
   ```

Ожидаемый результат обоих тестов: робот уверенно едет в обоих режимах, слушается пульта, failsafe работает.
