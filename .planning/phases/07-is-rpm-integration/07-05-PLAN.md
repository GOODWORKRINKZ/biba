<objective>
Wire the PI+FF closed-loop RPM controller into mode_standalone.c, replacing the
biba_ramp_t with an IRQ-driven ADC DMA state machine and biba_rpm_pi_step() calls.

This is the integration plan — it only modifies mode_standalone.c. All C modules
(zc_detector, adc_capture, rpm_pi) and proto/Python changes arrive from prior plans.

Architecture:
- Static state: biba_rpm_pi_state_t + volatile duty variables replace biba_ramp_t
- Init: start first DMA capture (LEFT channel)
- DMA IRQ callback: compute ZC → run PI → update volatile duty → start next capture
- Tick: read volatile duty → apply failsafe zero → call biba_hal_motor_pwm_*()
- Telemetry: set wheel_rpm_left/right_hz from s_rpm_pi_left/right.meas_ema

Purpose: Closes RPM-INT-01 and RPM-INT-02. After this plan the standalone firmware
builds and the PI loop runs on the robot.

Output: mode_standalone.c integrated; pio run -e rpico_rp2040_standalone succeeds;
human smoke test confirms wheel speed responds to RC commands.
</objective>

```c
static void on_adc_done(uint8_t channel, const uint16_t *buf, uint16_t n)
{
    float raw_hz = zc_freq_hz(buf, n, 10000u);

    if (s_adc_state == ADC_CAPTURING_LEFT) {
        /* ZC computed; update PI will happen after right channel */
        /* Store raw_hz temporarily — use s_rpm_pi_left's meas_ema as scratchpad */
        zc_ema_update(&s_rpm_pi_left.meas_ema, raw_hz, 0.0f);   /* pre-update EMA only */
        s_adc_state = ADC_CAPTURING_RIGHT;
        adc_capture_start_async(1u, 1024u, s_adc_buf, on_adc_done);  /* GP27 = IS_RIGHT = ch1 */
    } else if (s_adc_state == ADC_CAPTURING_RIGHT) {
        /* Both channels done. Run PI for both using target from last tick. */
        /* Note: target_hz is updated by tick; use volatile read is fine (same core0). */
        float raw_hz_right = raw_hz;
        float raw_hz_left  = s_rpm_pi_left.meas_ema;  /* EMA already updated above */

        /* Run PI step for both channels. Result written to volatile duty vars. */
        s_rpm_duty_left  = biba_rpm_pi_step(&s_rpm_pi_left,  &s_rpm_cfg,
                                             s_rpm_pi_left.meas_ema > 0.0f ?
                                             s_rpm_pi_left.meas_ema : 0.0f, raw_hz_left);
        s_rpm_duty_right = biba_rpm_pi_step(&s_rpm_pi_right, &s_rpm_cfg,
                                             s_rpm_pi_right.meas_ema > 0.0f ?
                                             s_rpm_pi_right.meas_ema : 0.0f, raw_hz_right);
        s_adc_state = ADC_CAPTURING_LEFT;
        adc_capture_start_async(0u, 1024u, s_adc_buf, on_adc_done);  /* GP26 = IS_LEFT = ch0 */
    }
}
```

Update on_adc_done() to read `s_target_hz_left` / `s_target_hz_right`. This avoids using meas_ema as a scratchpad.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| DMA IRQ → volatile duty vars | IRQ writes s_rpm_duty_left/right; tick reads them. Both on core0 — no mutex needed but volatile is required |
| Failsafe flag → PI reset | If PI reset is not called on failsafe, integral state persists across link loss — motor could lurch when link restores |
| Tick → on_adc_done target_hz | Tick writes s_target_hz_*; on_adc_done reads them. Both core0. Volatile required to prevent compiler optimization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05-01 | Safety | Failsafe edge misses PI reset | mitigate | Both failsafe AND disarm edges call biba_rpm_pi_reset() + zero s_rpm_duty_*. Redundant zeroing: tick also overrides duty=0 when failsafe flag active |
| T-07-05-02 | DoS | adc_capture_start_async re-entrant call | mitigate | on_adc_done only calls start_async after setting state to next phase; adc_capture_busy() would return false at that point (previous DMA finished, IRQ fired) |
| T-07-05-03 | Safety | PI integral survives mode switch | mitigate | biba_mode_standalone_init() called on mode entry — calls biba_rpm_pi_reset() which zeroes integral |
| T-07-05-04 | Safety | s_adc_buf overwrite mid-read | accept | Sequential L→R: DMA writes to s_adc_buf; on_adc_done reads it immediately in IRQ before starting next DMA. No concurrent access — single DMA channel in flight |
| T-07-05-05 | Safety | STANDALONE_RPM_MAX_HZ miscalibrated | accept | K = 940 Hz / 100% from Phase 06 FINDINGS. Error in K shifts setpoint proportionally; PI integral corrects SS error within spec (±10%). Out-of-range values do not cause unsafe operation — PI output clamped to [0,1] |
</threat_model>
