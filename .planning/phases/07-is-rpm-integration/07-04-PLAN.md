<!-- PI algorithm from PoC (is_rpm_poc_main.cpp cmd_rpmrun inner loop) — key operations -->
```
ff_duty   = (target_hz + ff_dead) / (ff_slope * 100.0f)  [clamped 0..1]
ki_active = (target_hz < ki_low_thresh) ? ki_low : ki
i_clamp_pos = 0.03f / (ki_active + 1e-6f)
i_clamp_neg = 0.01f / (ki_active + 1e-6f)
err       = target_hz - meas_hz
can_integrate = !(sat_high && err>0) && !(sat_low && err<0) && meas_hz > 50.0f
integral  += err * dt_s   [if can_integrate; then clamped to ±clamp]
p_term    = (meas_hz == 0.0f) ? 0.0f : kp * err   [clamped ±p_clamp]
i_term    = ki_active * integral
duty      = ff_duty + p_term + i_term
[stiction floor: if target>0 && 0 < duty < stiction_floor → duty = stiction_floor]
[clamp duty to [0.0, 1.0]]
```

  1. Phase 7: forward only — clamp `if (target_hz < 0.0f) target_hz = 0.0f`
  2. Call `zc_ema_update(&s->meas_ema, meas_raw_hz, target_hz)`; `float meas_hz = s->meas_ema`

  5. Asymmetric clamps: `float i_clamp_pos = 0.03f / (ki + 1e-6f); float i_clamp_neg = 0.01f / (ki + 1e-6f)`
  6. Error: `float err = target_hz - meas_hz`
  7. Anti-windup: `bool sat_high = s->prev_duty >= 0.999f; bool sat_low = s->prev_duty <= 0.001f; bool can_integrate = !(sat_high && err > 0.0f) && !(sat_low && err < 0.0f) && meas_hz > 50.0f`; if can_integrate: `s->integral += err * cfg->dt_s`; clamp integral to [-i_clamp_neg, i_clamp_pos]
  8. P term: `float p_term = (meas_hz == 0.0f) ? 0.0f : cfg->kp * err`; clamp p_term to [-cfg->p_clamp, cfg->p_clamp]
  9. I term: `float i_term = ki * s->integral`
  10. Duty: `float duty = ff_duty + p_term + i_term`
  11. Stiction floor: if `target_hz > 0.0f && duty > 0.0f && duty < cfg->stiction_floor` → `duty = cfg->stiction_floor`
  12. Clamp duty to [0.0f, 1.0f]; `s->prev_duty = duty; s->primed = true; return duty`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| target_hz / meas_raw_hz → rpm_pi_step | Values sourced from PoC calibration data and DMA IRQ ZC output; could be NaN or ±inf on hardware fault |
| integral accumulation | Unbounded growth if anti-windup fails; drives motor to full duty without measurement |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04-01 | DoS | integral runaway if meas_hz = NaN | accept | NaN propagates through float arithmetic → duty becomes NaN → hardware HAL clamps to [0,1] by cast; unlikely on RP2040 (no FP exceptions by default) |
| T-07-04-02 | Tampering | target_hz negative (reverse command) | mitigate | `if (target_hz < 0.0f) target_hz = 0.0f` at function entry (Phase 7 forward-only per CONTEXT.md D-01) |
| T-07-04-03 | DoS | NULL state or cfg pointer | mitigate | `if (s == NULL || cfg == NULL) return 0.0f` guard at function entry |
| T-07-04-04 | Tampering | duty NaN from ff_slope=0 division | mitigate | `if (cfg->ff_slope > 0.0f && target_hz > 0.0f)` guard before FF calculation; else ff_duty=0.0f |
</threat_model>
