```c
/* zc_freq_hz: A2 Sub-window Schmitt-trigger ZC detector.
 * n must be >= ZC_SUBWIN_K * 4. Splits buffer into ZC_SUBWIN_K blocks.
 * Per-block: finds min/max/mid/hyst, counts Schmitt transitions.
 * Returns (total_transitions/2) * sps / n.
 * Returns 0.0f if active_blocks < 2 OR total transitions < 2. */

/* zc_ema_update: EMA with validity gate (extracted from cmd_rpmrun).
 * hi = target_hz * 2.5f + 300.0f
 * If meas_raw in [ZC_MIN_VALID_HZ, hi] → EMA update (alpha=0.7)
 * If meas_raw == 0.0f → *ema *= 0.9f (slow decay)
 * else (noise spike) → hold EMA unchanged */
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ADC buffer → zc_freq_hz() | uint16_t[] from DMA — no bounds check in caller; n parameter must be validated |
| DMA IRQ → callback | IRQ context — callback runs on core0 ISR stack; must not block or allocate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01-01 | Tampering | zc_freq_hz n parameter | mitigate | Early return `0.0f` when `n < ZC_SUBWIN_K * 4u` is already in the algorithm — preserves safe return |
| T-07-01-02 | DoS | dma_irq_handler called with s_dma_ch=-1 | mitigate | Guard `if (s_dma_ch >= 0 && dma_channel_get_irq0_status(s_dma_ch))` before any action |
| T-07-01-03 | Tampering | adc_capture_start_async called while busy | mitigate | `if (s_dma_ch >= 0) return false;` prevents double-start race |
| T-07-01-04 | Information | EMA pointer NULL in zc_ema_update | accept | Internal function; caller (rpm_pi.c) always passes valid state pointer; NULL check adds overhead without safety benefit at this embedded tier |
</threat_model>
