Create scripts/is_rpm_calibrate.py as a standalone Python script. Follow the pyserial + argparse pattern from is_poc_capture.py exactly.

CLI arguments:
```
--port PORT          serial port (e.g. /dev/ttyACM0); omit in --dry-run mode
--wheel {left,right} which wheel (default: left)
--duties DUTIES      comma-separated duty points (default: 30,50,70,90)
--settle-ms MS       motor settle time per point (default: 3000)
--dry-run            skip serial; use synthetic K=10.13, dead=74.6 data for testing
--out-dir DIR        artifact directory (default: scripts/artifacts/calibration)
```

Main workflow (when not --dry-run):
1. Open serial port at 115200 baud, 2s boot wait
2. For each duty point in --duties:
   a. Print "→ duty %d%%: settle %dms..." to stderr
   b. Send `CALRUN <duty> <settle_ms>\n` to firmware
   c. Read response lines until "IS_HZ" line received (timeout 15s); parse: `is_hz = int(fields[2]) / 100.0`
   d. Prompt user: `input(f"  Enter tachometer Hz at {duty}%% (or ENTER to skip): ")`
   e. If user enters a valid float, append `{"duty_pct": duty, "is_hz": is_hz, "tach_hz": float(val)}` to points list
   f. If user skips, append `{"duty_pct": duty, "is_hz": is_hz, "tach_hz": None}`
3. Filter points where tach_hz is not None; need >= 3 valid points, else exit with error
4. Fit linear model: `coeffs = numpy.polyfit(duty_values, is_hz_values, 1)` → K_hz_per_pct = coeffs[0], dead_hz_raw = -coeffs[1]/coeffs[0]
5. Compute R²: predicted = numpy.polyval(coeffs, duty_values); ss_res = sum((is_hz - pred)**2); ss_tot = sum((is_hz - mean)**2); r_squared = 1 - ss_res/ss_tot
6. If r_squared < 0.95: print "WARNING: R²=%.3f < 0.95 — check hardware/filter" to stderr
7. Build output JSON with all required fields (see JSON format in context)
8. Save to `{out_dir}/{date}_{wheel}.json`; print path to stdout

--dry-run mode: generate synthetic points using K=10.13, dead=74.6 with ±2% random noise. Skip serial open and user prompts. Run full computation path to verify logic.

Import only stdlib + numpy + scipy (already available). Use `json`, `datetime`, `os`, `argparse`, `sys` from stdlib. Do NOT add new pip dependencies.
