Extend the biba_proto wire format with two uint16_t RPM fields carved from the
existing reserved[11] region, update the Python decoder atomically, and add speed
estimation to biba-controller. This plan is fully independent of the C module plans
(no zc_detector.h dependency) and can execute in parallel.

Purpose: Delivers the telemetry pipeline from RP2040 to Pi Python (RPM-INT-03).
The struct stays 48 bytes — no wire-level breaking change.

- biba_proto_telemetry_t extended; reserved[11] → reserved[7] + 2x uint16_t
- Python Telemetry dataclass gains wheel_rpm_left_hz + wheel_rpm_right_hz (float, Hz)
- config.py gains WHEEL_RADIUS_M + GEAR_RATIO env vars
- main.py gains wheel_rpm_to_mps() helper

In biba-controller/main.py, add a module-level helper function (after import block, before any class definitions):
```python
import math as _math
def wheel_rpm_to_mps(rpm_hz: float) -> float:
    """Convert IS-signal ZC frequency (Hz) to wheel linear speed (m/s).
    Returns 0.0 for rpm_hz <= 0 (no signal / motor stopped)."""
    if rpm_hz <= 0.0:
        return 0.0
    return (2.0 * _math.pi * rpm_hz * config.WHEEL_RADIUS_M) / config.GEAR_RATIO
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| wire bytes → Python Telemetry | Untrusted RP2040 data enters Python struct.unpack; malformed bytes could produce unexpected floats |
| env vars → config.py | WHEEL_RADIUS_M / GEAR_RATIO from environment — invalid values (zero, negative) could cause division by zero |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Tampering | reserved[7] bytes in proto | accept | Reserved bytes are zeroed by memset in biba_telemetry_collect; malformed non-zero values ignored by Python (field not read) |
| T-07-02-02 | DoS | GEAR_RATIO = 0.0 in config | mitigate | wheel_rpm_to_mps() guards `if rpm_hz <= 0.0 return 0.0`; add `if config.GEAR_RATIO == 0.0: return 0.0` guard in wheel_rpm_to_mps() |
| T-07-02-03 | Tampering | fields[20]/fields[21] overflow | accept | uint16 max 65535 → max 6553.5 Hz — well above motor range; no saturation risk |
| T-07-02-04 | Information | TELEMETRY_SIZE assert | mitigate | assert TELEMETRY_SIZE == 48 already in protocol.py catches struct drift at import time |
</threat_model>
