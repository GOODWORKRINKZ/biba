<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ADC callback → hint state | spec_left/spec_right results written by ADC IRQ; s_hint_hz_left/right are volatile floats accessed from main loop context |
| CRSF/command input → arm/disarm/OLON/OLOFF paths | These events trigger hint reset; no untrusted input changes hint to a harmful value since reset zeroes it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | Tampering | hint state persists across direction changes if reset missing | mitigate | D-B1: reset at all 14 DR-reset sites + per-channel at direction flip (Category D in Task 1) |
| T-10-07 | Tampering | hint fed from HINT_MEASURED result (circular) | mitigate | D8: update only when invalid_reason == INVALID_NONE; HINT_MEASURED (=6) excluded by the enum inequality |
| T-10-08 | Denial of Service | volatile float torn read on RP2040 (non-atomic 32-bit) | accept | RP2040 Cortex-M0+ single-cycle 32-bit loads are atomic for aligned floats; s_hint_hz_left is 4-byte aligned |
</threat_model>
