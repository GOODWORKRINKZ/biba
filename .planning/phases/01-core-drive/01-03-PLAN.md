  <behavior>
    All assertions use TEST_ASSERT_FLOAT_WITHIN(tolerance, expected, actual) unless boolean.
    Use tolerance 1e-5f for single-step calculations, 1e-4f for multi-step chains.

    Test 1 — test_init_starts_at_zero:
      biba_ramp_t r; biba_ramp_init(&r);
      r.current == 0.0f, r.hold_remaining_s == 0.0f

    Test 2 — test_reset_zeroes_running_state:
      Init, update to advance current to ~0.4f (target=1.0, dt=0.2), then biba_ramp_reset.
      r.current == 0.0f, r.hold_remaining_s == 0.0f

    Test 3 — test_dt_zero_returns_current_unchanged:
      Init; manually set r.current = 0.5f; call biba_ramp_update(&r, 1.0f, 0.0f).
      Returns exactly 0.5f; r.current still 0.5f. (Pitfall 1: first-tick guard)

    Test 4 — test_acceleration_toward_positive_target:
      Init (current=0); biba_ramp_update(&r, 1.0f, 0.1f).
      Expected: 0.2f (ACCEL_RATE=2.0 * dt=0.1 = 0.2 step from 0)

    Test 5 — test_deceleration_from_positive:
      Init; set r.current=1.0f; biba_ramp_update(&r, 0.0f, 0.1f).
      Expected: 0.8f (DECEL_RATE=2.0 * dt=0.1 = 0.2 step toward 0 from 1.0)

    Test 6 — test_direction_change_uses_reverse_decel_rate:
      Init; set r.current=0.5f; biba_ramp_update(&r, -1.0f, 0.1f).
      Expected: 0.45f (REVERSE_DECEL_RATE=0.5 * dt=0.1 = 0.05 step toward 0).
      NOT 0.3f (accel rate) or 0.4f (decel rate).

    Test 7 — test_direction_change_triggers_zero_hold:
      Init; set r.current=0.1f; call biba_ramp_update(&r, -1.0f, 1.0f) (large dt forces reach zero).
      After call: r.current == 0.0f AND r.hold_remaining_s is within 1e-4 of 0.15f.
      Call biba_ramp_update(&r, -1.0f, 0.05f) (tick during hold).
      Returns 0.0f still (hold_remaining_s > 0 → frozen at zero).

    Test 8 — test_clamp_output_to_unit:
      Init; set r.current=0.9f; call biba_ramp_update(&r, 1.0f, 10.0f) (enormous dt).
      Returns exactly 1.0f — never exceeds 1.0f.
      Also: init; set r.current=-0.9f; call biba_ramp_update(&r, -1.0f, 10.0f).
      Returns exactly -1.0f.
  </behavior>
